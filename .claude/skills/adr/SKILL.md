# ADR Manager

Управляет архитектурными решениями в директории `.claude/adr/`.

## Структура директории
```
.claude/adr/
  ADR-001-short-title.md
  ADR-002-another-decision.md
  README.md   <- авто-генерируемый индекс, не редактировать вручную
```

## Формат файла ADR

Имя файла: `ADR-NNN-kebab-case-title.md` (номер дополняется нулями до 3 цифр)

Шаблон:
```markdown
# ADR-NNN: Заголовок

**Статус:** Принято
**Дата:** YYYY-MM-DD

## Контекст
Почему это решение потребовалось. Какую проблему решает?

## Решение
Что именно решили. Конкретно.

## Последствия
+ Положительный результат
- Компромисс или риск
```

Допустимые статусы: `Принято` / `Отклонено` / `Заменено ADR-NNN` / `Устарело`

---

## Workflow: Создать новый ADR

1. Сканировать `.claude/adr/` на `ADR-*.md`, найти макс NNN, увеличить на 1. Начинать с 001 если файлов нет.
2. Имя файла: `ADR-NNN-kebab-title.md` (строчные, дефисы)
3. Если детали не предоставлены — запросить: заголовок, контекст, решение, последствия
4. Сегодняшняя дата, статус = `Принято` по умолчанию
5. Показать готовый файл пользователю, подтвердить перед записью
6. Записать файл, затем сразу обновить индекс

---

## Workflow: Обновить индекс (README.md)

Сканировать все `ADR-*.md` в `.claude/adr/`, из каждого извлечь:
- Номер: из имени файла `ADR-NNN-...`
- Заголовок: из `# ADR-NNN: <заголовок>` — часть после двоеточия
- Статус: из строки `**Статус:**`
- Дата: из строки `**Дата:**`

Перезаписать `.claude/adr/README.md` целиком, сортировка по номеру:
```markdown
# Architecture Decision Records

| №       | Решение                          | Статус           | Дата       |
|---------|----------------------------------|------------------|------------|
| ADR-001 | [Заголовок](ADR-001-file.md)     | Принято          | 2026-01-15 |
| ADR-002 | [Заголовок](ADR-002-file.md)     | Заменено ADR-003 | 2026-02-01 |
```

---

## Workflow: Валидация

Проверить каждый `ADR-*.md` на наличие: заголовка, Статуса (допустимое значение),
Даты (YYYY-MM-DD), непустых секций Контекст / Решение / Последствия.
Вывести по каждому файлу: OK или список проблем.

---

## Workflow: Показать список ADR
```
ADR-001  Принято   2026-01-15  Заголовок решения
ADR-002  Устарело  2026-02-10  Другое решение
```

---

## Примечания

- Создать `.claude/adr/` если не существует
- Всегда подтверждать с пользователем перед записью на диск
- README.md только перегенерируется целиком, никогда не патчится
- Заменено всегда с номером: `Заменено ADR-007`
